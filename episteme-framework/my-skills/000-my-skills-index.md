# my-skills (this folder)

Index of folders and _docs. One _doc per folder defines the schema for that folder’s content.

| Folder | _Doc | Purpose |
|--------|------|---------|
| **skill-definition** | `_skill-definition.md` | Schema for one skill: name, description, when to use, instructions. Skill = folder (folder name = skill name) with content; meta-docs for all skills (especially imported). Check the website for the standard. |
| **skill-definition/skill-standard** | `_skill-standard.md` | Rules and regulations, folder design, structure design rules for a SKILL. Check the website for the standard. |
| **skill-definition/skill-example-files** | `_skill-example-files.md` | Example skills as folders (e.g. FLIP_WIDGET): abstract, agnostic; conform to skill-definition and skill-standard. |
| **imported-skills** | `_imported-skills.md` | Meta-doc per imported skill: location, risk assessment (security, trust, dependencies, conflicts; summary; alternatives), conversation notes (AGENT with other ENTITIES). Skills from website or other sources. |
| **episteme-skills** | `_episteme-skills.md` | Framework-specific skills: one folder per skill (folder name = skill name); small to small-medium each; complex skills = constellation of simpler skills, singleness of purpose. Necessary to navigate the framework; not likely for use outside this context. |
